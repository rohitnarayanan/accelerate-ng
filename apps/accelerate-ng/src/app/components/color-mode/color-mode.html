ColorMode Component

<angb-color-mode> — Color scheme switcher

The Color Mode component lets users switch between Light, Dark, and Auto themes. It can render as a floating button or a dropdown inside header/footer menus. State is persisted via UserSettingsService under the key colorMode.

Examples

1) Floating button (draggable)

A compact button you can position anywhere; clicking cycles the theme.

Show code 

    <angb-color-mode [mode]="'button'" [options]="" />
2) Dropdown (for Header/Footer menus)

A dropdown menu entry that exposes Light, Dark, and Auto with an active check mark.

Rendered by Header/Footer when using a navbar item of type color-mode.

API

Inputs
Name Type Default Description
mode 'button' | 'menu' 'button' Render style: floating button or dropdown menu entry.
[options] ColorModeOptions — Component configuration (see below).
Options: ColorModeOptions
Option Type Default Description
id string 'colorModeToggle' Element id.
buttonClass string 'btn-warning' Extra classes for the floating button.
menuClass string — Classes applied to the dropdown toggle when rendered in menus.
menu Omit<SubMenu, 'items'> — Menu presentation options like type ('dropdown' | 'dropup') and classes.
Behavior
  • Available modes: light, dark, auto.
  • In button mode, clicking the button cycles modes.
  • In menu mode, choose the desired mode from the dropdown list.
  • Active mode is indicated with a check mark and the toggle icon updates accordingly.
  • Selection is persisted via UserSettingsService using the key colorMode.

Quick Usage

Direct in a template
<!-- Floating button -->
<angb-color-mode [mode]="'button'" [options]="" />

<!-- Dropdown (e.g., place in header/footer containers) -->
<angb-color-mode [mode]="'menu'" [options]="" />
Via Header/Footer configuration (from app.ts)
